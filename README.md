# 고양은평선 식사동 연장 데이터 플랫폼

> **siksaline.kr**
> 감정이 아닌 데이터로. 고양 북동부의 교통복지와 균형발전을 위한 정치적 중립 시민 플랫폼.

식사동 주민의 교통 접근성 개선과 고양시 북동부 균형발전의 필요성을 **공공데이터**로
분석·시각화하고, 주민 서명을 통해 정책을 제안합니다.

## 기술 스택

- **Next.js 16** (App Router) · **React 19** · **TypeScript**
- **Tailwind CSS v4** (`@theme` 토큰 디자인 시스템)
- **Pretendard** 서체 (Noto Sans KR 폴백)
- 차트: 외부 의존성 없는 **순수 SVG 컴포넌트** (`src/components/charts.tsx`)
- (예정) **Supabase** 서명 저장·데이터 적재 / **지도 API** 노선 레이어 / **공공데이터 API**

## 메뉴 구성

| 경로 | 메뉴 | 내용 |
| --- | --- | --- |
| `/` | 홈 | 오늘의 데이터 대시보드, 핵심 요약 |
| `/progress` | 진행현황 | 추진 단계, 정책 타임라인, 공약 추적기 |
| `/route-map` | 노선도 | 연장 개념도, 예상 역세권 분석 |
| `/transit` | 교통 데이터 | 인구·철도 접근성, 출퇴근 시뮬레이터, 확보 예정 지표(혼잡도·배차·경제효과·교통 소외지수) |
| `/petition` | 주민 서명 | 온라인 서명 폼, 참여 현황, 주민 의견 |
| `/news` | 뉴스 | 관련 보도 모음 |
| `/resources` | 자료실 | 원자료(CSV/PDF) 다운로드, 공개 API 예정 |


## 데이터에 대하여

지표는 **공개 공공데이터·언론 보도에 근거**하며 각 항목에 출처를 표기합니다.
검증된 값(인구·서명 규모·노선 사실·정책 일지 등)만 수치로 표시하고,
실측이 필요한 지표(버스 혼잡도·경제효과·교통 소외지수 등)는 임의값 대신
**"자료 확보 중"**으로 투명하게 표시합니다.
모든 데이터는 `src/data/` 에 모듈로 분리되어 있어 공공데이터 API/Supabase 연동으로 교체할 수 있습니다.

**확보 예정(실데이터 연동 대상):** 버스 혼잡도·배차(TAGO/경기버스정보), 목적지별 통행시간(KTDB),
경제효과·교통 소외지수(공인 방법론), 지역별 서명 집계, 인구 시계열(KOSIS).

- `src/data/overview.ts` — 비전·철학·오늘의 데이터·기능
- `src/data/transit.ts` — 인구·철도 접근성·통행시간·시뮬레이터·확보 예정 지표(혼잡도·배차·경제효과·소외지수)
- `src/data/route.ts` — 노선·역·역세권
- `src/data/civic.ts` — 진행단계·타임라인·서명·뉴스·자료실

환경변수는 `.env.example` 참고.

## 원칙

- **감정이 아닌 데이터** · **집값이 아닌 교통복지** · **정치적 중립** · **객관적 공공데이터**
- 모든 지표는 출처와 기준시점을 표기하고 원자료를 공개합니다.
