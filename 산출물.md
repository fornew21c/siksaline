## 4. 핵심 기능

### 온라인 주민 서명 (실서명)
- **이메일 인증 방식** (Supabase Auth 매직링크): 이름·지역·이메일 입력 → 인증 메일 링크 클릭 → 서명 완료
- **명단 완전 비공개**: RLS로 개별 행 읽기 차단, 공개되는 것은 **집계 숫자만**(`signature_count` RPC)
- **중복 차단**: 이메일당 1건 (upsert)
- **위조 방지**: 서버가 access_token 재검증 후 저장
- **실시간 집계**: 서명 즉시 반영 + 30초 ISR

### 현수막 온라인 동기화
- 오프라인 현수막과 동일 슬로건·요구를 홈/서명 페이지에 **현수막 스타일 구역**으로 표시 (검정+형광노랑+빨강)

### Open Graph (링크 공유 카드)
- 브랜드 OG 이미지 `public/og.png` (1200×630, 현수막 톤)
- `og:*` + `twitter:summary_large_image` 메타 (카카오톡·페북·트위터 큰 카드)

### 기타
- 출퇴근 시뮬레이터, 순수 SVG 차트, 반응형(모바일/태블릿/데스크톱), 정치적 중립 명시

### ✅ 검증된 실데이터 (출처 표기)
| 지표 | 값 | 출처 |
|---|---|---|
| 식사·풍동 일대 주민 | 약 8만 명 (입주 시 10만+) | 추진모임·보도 |

| 통근 개선 | 서울 도심 50분대→20분대 | 한국경제 보도 |

### ⏳ 자료 확보 중 (임의값 대신 "자료 확보 중" 표기)
버스 혼잡도(TAGO), 배차, 목적지별 통행시간(KTDB), 경제효과·교통 소외지수(공인 방법론), 서울 통근 비율(통계청), 지역별 서명 집계, 인구 시계열(KOSIS)

### 데이터 위치 (교체 지점)
- `src/data/overview.ts` — 비전·철학·오늘의 데이터·캠페인·슬로건
- `src/data/transit.ts` — 인구·통근·철도·확보예정
- `src/data/route.ts` — 노선·역·팩트
- `src/data/analysis.ts` — 시뮬레이터·확보예정
- `src/data/civic.ts` — 진행단계·타임라인·서명·뉴스·자료실

## 6. 인프라 · 운영 가이드

### 배포
- `git push origin main` → Vercel 자동 배포
- 수동: `vercel --prod`

### 환경변수 (Vercel + 로컬 `.env.local`)
```
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...        # 서버 전용 · 비공개
```
(Vercel production/preview/development 3개 환경 모두 등록됨)

### Supabase DB 설정
- 스키마/RLS/집계 함수: `supabase-setup.sql` (SQL Editor에서 1회 실행 완료)
- 테이블: `public.signatures` (email, name, region, verified_at, created_at)

### 서명 명단 확인 방법
- Supabase 대시보드 → Table Editor → `signatures` (관리자만, 공개 X)
- SQL: `select public.signature_count();` (집계), `select * from public.signatures;` (명단)

### 로컬 실행
```
npm install
npm run dev      # http://localhost:3000
npm run build
```
